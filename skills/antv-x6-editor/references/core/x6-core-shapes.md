## Overview of Built-in Shapes in X6 3.x

> Verified from `src/shape/index.ts`: 10 shapes are exported. The `Shape` namespace **only** contains these 10 shapes, without `Cylinder / Diamond / Group / Hexagon`.

| Class Name | Shape String | Tag Name (Body) | Use Cases |
|------------|--------------|-----------------|------------|
| `Rect`     | `'rect'`     | `<rect>`        | General rectangle nodes, flowchart steps (most commonly used) |
| `Circle`   | `'circle'`   | `<circle>`      | Status nodes, start/end points |
| `Ellipse`  | `'ellipse'`  | `<ellipse>`     | Elliptical nodes, emphasis |
| `Polygon`  | `'polygon'`  | `<polygon>`     | Polygons (rhombus, hexagon, pentagram, etc.) |
| `Polyline` | `'polyline'` | `<polyline>`    | Polyline (open path, not closed) |
| `Path`     | `'path'`     | `<path>`        | Any SVG path (icons, freeform shapes) |
| `Image`    | `'image'`    | `<image>`       | Image nodes (icons) |
| `TextBlock`| `'text-block'`| `<rect>` + foreignObject | Multi-line text blocks (HTML layout) |
| `HTML`     | (requires registration) | `foreignObject` | Rich HTML nodes |
| `Edge`     | `'edge'`     | Dual `<path>`   | Default edge type |

### Several Naming Differences That Are Easy to Overlook

- In the shape string, **`text-block`** uses a hyphen, not camel case
- `Shape.HTML` is a **class**, but you must first call `Shape.HTML.register({ shape: 'xxx', ... })` to register a named shape before you can `addNode({ shape: 'xxx' })`; HTML cannot directly `addNode({ shape: 'html' })`
- `Edge` is the default edge shape string `'edge'`, and almost all `graph.addEdge({...})` default to inheriting it, so there’s no need to explicitly declare it

## Common Default attrs (Verified from `src/shape/base.ts`)

All node shapes generated by `createShape` (except `path` / `text-block`) share:

```javascript
// BaseBodyAttr
attrs.body = { fill: '#ffffff', stroke: '#333333', strokeWidth: 2 }

// BaseLabelAttr (written to attrs.text, selector name is 'text')
attrs.text = {
  fontSize: 14, fill: '#000000',
  refX: 0.5, refY: 0.5,
  textAnchor: 'middle', textVerticalAnchor: 'middle',
  fontFamily: 'Arial, helvetica, sans-serif',
}
```

The `label` field (top-level) will be automatically inserted into `attrs/text/text` by `propHooks`. That is:

```javascript
graph.addNode({ shape: 'rect', label: 'Hello' });
// Equivalent to
graph.addNode({ shape: 'rect', attrs: { text: { text: 'Hello' } } });
```

> Note: **The text selector for built-in shapes in attrs is `text`, not `label`**. The selector in `util.ts:getMarkup` is written as `'label'`, but `Base.config` registers the attrs key as `text`. Both can be matched, but **it is recommended to use `attrs.text`** (consistent with the base class).

## 1. Rect (Rectangle, Most Commonly Used)

**Source Code (`src/shape/rect.ts`)**:

```javascript
attrs: { body: { refWidth: '100%', refHeight: '100%' } }
```

`refWidth: '100%'` / `refHeight: '100%'` allows the `<rect>` to automatically fill the node's `width / height`. Rounded corners are set via `rx / ry`.

```javascript
graph.addNode({
  shape: 'rect',
  x: 100, y: 60, width: 120, height: 40,
  label: 'Rectangle',
  attrs: {
    body: { stroke: '#8f8f8f', strokeWidth: 1, fill: '#fff', rx: 6, ry: 6 },
  },
});
```

## 2. Circle

**Source Code (`src/shape/circle.ts`)**:

```javascript
attrs: { body: { refCx: '50%', refCy: '50%', refR: '50%' } }
```

Via `refR: '50%'`, the radius automatically takes `min(width, height) / 2`. **To obtain a perfect circle, `width` and `height` must be equal**.

```javascript
graph.addNode({
  shape: 'circle',
  x: 60, y: 100, width: 60, height: 60,   // width and height must be equal
  label: 'Start',
  attrs: { body: { fill: '#f6ffed', stroke: '#52c41a', strokeWidth: 2 } },
});
```

## 3. Ellipse

**Source Code (`src/shape/ellipse.ts`)**:

```javascript
attrs: { body: { refCx: '50%', refCy: '50%', refRx: '50%', refRy: '50%' } }
```

`width / height` do not need to be equal; half of each will be automatically taken as `rx / ry`.

```javascript
graph.addNode({
  shape: 'ellipse',
  x: 200, y: 80, width: 120, height: 60,
  label: 'Process',
  attrs: { body: { fill: '#e6f7ff', stroke: '#1890ff', strokeWidth: 2 } },
});
```

## 4. Polygon (Including Rhombus / Hexagon)

**Source Code (`src/shape/poly.ts`)**: The `points` field is written to `attrs/body/refPoints` by `propHooks`.

> `refPoints` is a custom X6 attribute: accepts **percentage** coordinates as input and automatically scales according to the node's BBox.

### Method 1: Top-level `points` field (recommended)

Supports the following three formats:
- `'x1,y1 x2,y2 ...'` string (standard SVG points)
- `[[x1,y1], [x2,y2]]` array of arrays
- `[{x,y}, {x,y}]` array of objects

```javascript
// Rhombus (decision node): percentage coordinates
graph.addNode({
  shape: 'polygon',
  x: 100, y: 100, width: 120, height: 60,
  label: '?',
  points: '60,0 120,30 60,60 0,30',           // ← string format
  attrs: { body: { fill: '#fff', stroke: '#8f8f8f' } },
});

// Hexagon: array format
graph.addNode({
  shape: 'polygon',
  x: 240, y: 100, width: 120, height: 60,
  points: [[30, 0], [90, 0], [120, 30], [90, 60], [30, 60], [0, 30]],
  attrs: { body: { fill: '#fff7e6', stroke: '#fa8c16' } },
});
```

### Method Two: Directly Write `attrs.body.refPoints`

```javascript
attrs: { body: { refPoints: '60,0 120,30 60,60 0,30' } }
```

> ⚠️ Do not write both the top-level `points` and `attrs.body.refPoints`, as they will conflict (the top-level `propHook` will override).

## 5. Polyline (Broken Line, Open Path)

Shares the `Poly` base class with Polygon (`src/shape/polyline.ts`), with consistent usage. The difference lies in that `<polyline>` is **not closed**.

```javascript
graph.addNode({
  shape: 'polyline',
  x: 100, y: 100, width: 120, height: 60,
  points: '0,60 30,0 60,60 90,30 120,60',
  attrs: { body: { fill: 'none', stroke: '#1890ff', strokeWidth: 2 } },
});
```

## 6. Path (Arbitrary SVG Path)

**Source Code (`src/shape/path.ts`)**: Unlike other shapes, the path node has 3 markup elements (`bg` transparent background + `body` actual path + `label` text). The top-level `path` field is written to `attrs/body/refD` by propHook.

```javascript
graph.addNode({
  shape: 'path',
  x: 100, y: 100, width: 120, height: 80,
  path: 'M 60 0 L 120 80 L 0 80 Z',        // ← Triangle path
  label: 'Triangle',
  attrs: { body: { stroke: '#722ed1', strokeWidth: 2, fill: '#f9f0ff' } },
});
```

- `refD` scales with the BBox like `refPoints`, so path coordinates can be written relative to node dimensions
- For path nodes, `attrs.body.fill` defaults to `none`, `stroke` to `#000`, and `strokeWidth: 2` (different from other shapes!)
- A transparent hit area is required for hover/click events: path nodes come with a built-in `bg` that has `pointerEvents: 'all'` set

## 7. Image

**Source Code (`src/shape/image.ts`)**: The internal selector is **`image`** instead of `body`, because `createShape` overrides the default with `selector: 'image'`. Provides 3 shortcut fields: `imageUrl` / `imageWidth` / `imageHeight`.

```javascript
graph.addNode({
  shape: 'image',
  x: 60, y: 60, width: 60, height: 60,
  // Shortcut fields (recommended)
  imageUrl: 'https://gw.alipayobjects.com/zos/antfincdn/FLrTNDvlna/antv.png',
  // Equivalent to: attrs: { image: { 'xlink:href': '...' } }
});

// Also supports full attrs syntax
graph.addNode({
  shape: 'image',
  x: 200, y: 60, width: 80, height: 80,
  attrs: {
    image: {
      'xlink:href': 'https://gw.alipayobjects.com/zos/antfincdn/FLrTNDvlna/antv.png',
      width: 80, height: 80,
    },
  },
});
```

> ⚠️ Syntax like `attrs.body.fill` is **invalid** for image nodes——image nodes do not have a `body` selector at all; all styles must be written under `attrs.image`.

## 8. TextBlock (Multi-line Text Block)

**Source Code (`src/shape/text-block.ts`)**:
- When the browser supports `foreignObject` (in most cases), the body is `<rect>`, and the text is rendered via `foreignObject > div`, **automatically supporting line breaks**
- Falls back to `<rect> + <text>` when not supported
- The top-level `text` field is written to `attrs/label/text` by propHook

```javascript
graph.addNode({
  shape: 'text-block',
  x: 100, y: 100, width: 200, height: 80,
  text: 'This is a long multi-line text\nthat will automatically wrap and center',
  attrs: {
    body: { fill: '#fff', stroke: '#8f8f8f', strokeWidth: 1, rx: 6, ry: 6 },
    label: { style: { fontSize: 13, color: '#262626' } },
  },
});
```

> In `text-block`, label styles must be placed in `attrs.label.style` (HTML inline style), not SVG attributes. Use `style.color` instead of `fill` to set the color.

## 9. HTML (Rich HTML Nodes)

`Shape.HTML` must be registered first and cannot be directly added as a node. **For a complete guide, all business templates (cards / forms / status badges), effect re-rendering mechanisms, and migration instructions for the deprecated X6 2.x API `Graph.registerHTMLComponent`, refer to [`core/x6-core-html-shape.md`](./x6-core-html-shape.md)**. Here is the simplest method of invocation:

```javascript
import { Graph, Shape } from '@antv/x6';

Shape.HTML.register({
  shape: 'my-html',
  width: 200, height: 80,
  effect: ['data'],
  html(node) {
    const div = document.createElement('div');
    div.style.cssText = 'width:100%;height:100%;border:1px solid #8f8f8f;padding:8px;';
    div.innerHTML = `<b>${node.getData()?.title || ''}</b>`;
    return div;
  },
});

graph.addNode({
  shape: 'my-html',
  x: 100, y: 60,
  data: { title: 'Hello HTML' },
});
```

> ⚠️ X6 3.x **does not** have the `Graph.registerHTMLComponent` API (deprecated in X6 2.x). All HTML nodes must be registered uniformly through `Shape.HTML.register`.

## 10. Edge (Default Edge)

`shape: 'edge'` is the default shape for `graph.addEdge`, and it rarely needs to be explicitly declared. The markup consists of two paths (a transparent wrap used as a hit-area + the actual line). For more details, refer to `core/x6-core-edge.md` and `core/x6-core-markup.md`.

```javascript
graph.addEdge({
  source: a, target: b,
  // shape: 'edge',   // Can be omitted
  attrs: {
    line: { stroke: '#8f8f8f', strokeWidth: 1, targetMarker: 'classic' },
  },
});
```

## Selection Guide

| Business Intent | Recommended Shape | Key Fields |
|----------|-----------|----------|
| General Process Steps | `rect` + rx/ry rounded corners | `attrs.body.{fill,stroke,rx,ry}` |
| Start / End Nodes | `circle` | `width === height` |
| Judgment / Decision | `polygon` (diamond) | `points: '60,0 120,30 60,60 0,30'` |
| Resource / Database | `path` | `path: 'M ...'` |
| Nodes with Icons | `image` (simple) / `Shape.HTML` (complex) | `imageUrl` / `html(cell)` |
| Multi-line Text Description | `text-block` | `text: '...'` |
| Rich UI (Buttons, Cards, Forms) | `Shape.HTML.register` | `effect: ['data']` |
| Edge | `edge` (default, can be omitted) | `attrs.line.*` |

## Common Errors and Fixes

### ❌ Using Non-Existent Shapes

```javascript
// Error: X6 3.x does not have Cylinder / Diamond / Group / Hexagon
graph.addNode({ shape: 'cylinder', ... });   // ❌ Throws "shape not found"
graph.addNode({ shape: 'diamond',  ... });   // ❌
new Shape.Group();                            // ❌ Cannot read properties of undefined

// Correct
graph.addNode({ shape: 'polygon', points: '60,0 120,30 60,60 0,30', ... });  // Diamond
graph.addNode({ shape: 'path',    path: 'M 0 20 Q 60 0 120 20 ...', ... });  // Cylinder using path
```

### ❌ Missing `points` in Polygon

```javascript
// Error: Without `points`, the <polygon> element cannot obtain any vertices, rendering it empty
graph.addNode({ shape: 'polygon', x: 100, y: 100, width: 120, height: 60 });

// Correct
graph.addNode({
  shape: 'polygon',
  x: 100, y: 100, width: 120, height: 60,
  points: '60,0 120,30 60,60 0,30',
});
```

### ❌ Image Node Styles Written to `attrs.body`

```javascript
// Incorrect: The selector for the image node is 'image', not 'body'
graph.addNode({
  shape: 'image',
  x: 60, y: 60, width: 60, height: 60,
  attrs: { body: { 'xlink:href': '...' } },   // ❌ Does not take effect
});

// Correct
graph.addNode({
  shape: 'image',
  x: 60, y: 60, width: 60, height: 60,
  imageUrl: 'https://...',                    // ✅ Shortcut field
});
// Or
graph.addNode({
  shape: 'image',
  x: 60, y: 60, width: 60, height: 60,
  attrs: { image: { 'xlink:href': 'https://...' } },
});
```

### ❌ Missing `path` in Path Node

```javascript
// Error: Only renders a transparent background <rect bg>, no path is visible
graph.addNode({ shape: 'path', x: 100, y: 100, width: 120, height: 80 });

// Correct: `path` field is required
graph.addNode({
  shape: 'path',
  x: 100, y: 100, width: 120, height: 80,
  path: 'M 60 0 L 120 80 L 0 80 Z',
});
```

### ❌ Circle Node width !== height

```javascript
// Error: Rectangle width/height will cause refR: '50%' to take the smaller value, resulting in a smaller node and centered blank display
graph.addNode({ shape: 'circle', x: 0, y: 0, width: 100, height: 60 });

// Correct: Circle maintains equal width and height
graph.addNode({ shape: 'circle', x: 0, y: 0, width: 60, height: 60 });
// Alternatively, use ellipse
graph.addNode({ shape: 'ellipse', x: 0, y: 0, width: 100, height: 60 });
```

### ❌ HTML Node Directly Added Without Registration

```javascript
// Error: 'html' is not a default available shape string
graph.addNode({ shape: 'html', html: '<div>x</div>' });   // ❌ Throws shape not found

// Correct: Register a named HTML shape first
Shape.HTML.register({
  shape: 'card',
  html(node) { /* ... */ },
});
graph.addNode({ shape: 'card', x: 0, y: 0, width: 200, height: 80 });
```

### ❌ TextBlock Uses SVG Attributes to Set Font Color

```javascript
// Incorrect: Inside foreignObject is an HTML <div>, fill is invalid
attrs: { label: { fill: '#f00' } }

// Correct: Use HTML inline style
attrs: { label: { style: { color: '#f00', fontSize: 14 } } }
```
